# CLAUDE.md — agentdesk

Working brief for Claude Code. Read this first each session, then `README.md` (full spec) and `DECISIONS.md` (ADRs with rationale).

## What this is

An AI-powered ITSM service desk (mini ServiceNow) — a **portfolio project**. A tool-less **router agent** classifies each query and hands off to one of three specialists:

- **knowledge** — query expansion, hybrid RAG (pgvector + FTS, reciprocal rank fusion), citations
- **fulfillment** — reads user assets, pre-fills catalog orders, human-in-the-loop approval for orders > $500
- **incident** — summarize, dedup via ticket embedding similarity, create/link tickets

**Stack:** OpenAI Agents SDK (models via LiteLLM) · FastAPI · Streamlit · Postgres+pgvector (relational + vector + FTS in one DB) · Redis (embedding/semantic/response caches) · Langfuse · custom eval harness in GitHub Actions. Later: MCP server (M8), Neo4j Graph-RAG (M9).

**Key invariants (see DECISIONS.md):** tools are the *only* DB access path; router carries no tools; semantic cache runs before any agent and only for read-only intents; short-term memory = SDK sessions in Postgres, long-term = hand-rolled `user_facts` (inject at session start, extract at end).
